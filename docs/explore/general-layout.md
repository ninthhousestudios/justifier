i want to think about names -- i dont really like "lesson".
each of the 00/01, etc. sections would obviously be "modules" but i want to see if we
can think of something more creative. i want the user to have fun exploring, not to feel
like they have to "learn" something or "get it right" or "do it perfectly". it is about
the experience of trying something new and opening yourself to a new world

i like circles so i want to experiment with having most things in circles

# main explore screen

each lesson has a circular emblem with the title and a short description underneath
tapping takes the user to the lesson itself

# the lessons

the focus for this app is listening and feeling, understanding ji in your ears/body,
both by listening and production

the main lesson page will be a navigator

the primary circle in the middle will be the listening/production exercise for the
module
above this perhaps a short description of this lesson and what it does

then there will be surrounding circles

"Nerd Out!" -> this has detailed technical information about what is happening and why
"Examples" (need a better name) -> has experiential things, like examples of music using
these concepts

# progress

there should be someone way to track "progress", but this is not required.
progress tracking is entirely driven by the user and up to the user; if they want to
mark what they have done, or mark something they dont understand well and want to return
to, etc. they can, but none of this is required. it is all user-driven; accords with how
they want to use the app
